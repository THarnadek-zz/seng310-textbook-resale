Requirements

A project for SENG 310 Spring 2017

Requirements

Requirement # 1: User can create a new listing

Description

The user can create a new listing. The system must then share this listing with other users.

Rationale

This is a core functionality - failure to share the listing would mean that no one would see it.

Fit Criterion

The listing should be visible to other users once it has been submitted.

Scenario

A user wants to sell their book. They use the application to make a listing and share it with other users.

 

Requirement # 2: User can browse listings

Description

The user can look through existing listings. The system should have some form of filtering so that the user can narrow their search.

Rationale

The user needs to be able to see what books are available.

Fit Criterion

The system shall allow users to view all existing listings.

Scenario

Joshua wants to buy a textbook for their Philosophy class. They manually browse the listings, and are eventually able to locate the book that they need.

 

Requirement # 3: User can search listings

Description

The seller can attach tags to their listing in order to filter buyer’s searches

Rationale

The buyer can have a specific criteria for the book and point of purchase, and wants to be able to find a post that suits his needs.

Fit Criterion

When the user enters search criteria, the system shall narrow the listings down and display only those books that fit the criteria.

Scenario

Ashley is looking for a SPAN 100 textbook but her price range is $50-$60. She wants the edition 3 for the textbook since the older editions are not permitted. She wants to find the textbook that fits her criteria and be able to search for these tags.

 

Requirement # 4: Buyer can contact the seller

Description

Buyer can contact the seller to discuss purchase

Rationale

In order to buy or sell a book, contact has to be made between the buyer and seller.

Fit Criterion

When the seller wants to inquire about a book, the system facilitates anonymous contact.

Scenario

Frank is interested in purchasing a CSC book that Robbie has listed. Since the posting is still open, he leaves a comment or message for Robbie, noting that he is interesting in purchasing the textbook. An Email or direct notification is sent to Robbie notifying him that his listing has been commented on.

 

Requirement # 5: Seller can update status of the sale

Description

Status bar will display “Pending, Available, or Sold” status for the textbook post

Rationale

This will save time for the buyer and narrow their search to available items.

Fit Criterion

Seller will update the status of the book’s availability

Scenario

John is interested to purchase a Calculus textbook that Dan posted on the app. He wants to see if it is still available, and he doesn’t want to waste his time messaging Dan if the book is no longer available. He is able to check the status bar, and he can see that the sale is pending. He decides to bookmark the post and keep an eye on it incase it becomes available.

Requirement # 7: Commercial Users can post bulk listings

Description

Commercial sellers can make Bulk listings for their products

Rationale

This will allow for commercial users to easily show off their selection without making too many listings

Fit Criterion

When a Commercial user needs to advertise several copies of the same book, they can easily make a bulk listing

Scenario

The Local used bookstore has several copies of an ENGR textbook. Since they possess multiple copies, they set up a listing for the ENGR textbook, with a quantity tab.

 

Requirement # 8: The User can delete their listings

Description

Users can easily delete their listings if they want.

Rationale

Users that have sold their books outside of the app or found that they still need them will be able to easily take down their listing

Fit Criterion

When a User has either sold their book, or sold their books outside of the app.

Scenario

Joshua originally wanted to sell his Math 100 Textbook, but found after he posted his listing that he has to take Math 101, which requires the same textbook. He then quickly and easily deletes his listing.

 

Requirement # 9: The User can rate a Seller

Description

Buyers can post a rating on a Seller’s profile that they have interacted with regarding a sale

Rationale

Buyers can rate their Seller out of 5 stars based on the interaction and transactional experience with the Seller.

Fit Criterion

After a Buyer has interacted with the Seller, they can indicate their experience with the sale

Scenario

After Robbie sold his CSC book to Frank, Frank decides that the whole experience with him was good, and rates Robbie 5 stars.
